# AGENTS.md

## 基本方針

- 回答と作業報告は日本語で行う。
- 推測で実装しない。不明点や矛盾がある場合は明示する。
- 既存設計を尊重し、依頼範囲外の大規模変更は行わない。
- 既存の未コミット変更を消去・上書きしない。
- Git の commit・push は明示指示がある時だけ行う。
- 追加ライブラリは必要最小限にする。

## 作業手順

1. 現状確認
2. 修正計画の提示
3. 必要最小限の実装
4. テストまたは動作確認
5. `plan.md` 更新
6. 日本語での作業報告

## 見積計算ルール

- 明細金額は `数量 × 単価` の自動計算とする。
- 金額、税抜小計、消費税、合計などの自動計算欄は直接編集不可にする。
- Delete キー等で数量や単価を空にしても、計算ロジック自体は消えない構造にする。
- 通常値引きは税抜処理とする。
- 端数調整は消費税計算後の税込総額から引く。
- 計算順は `税抜合計 - 値引き = 小計 + 消費税 - 端数調整 = 合計（税込）` とする。

## PDF・Excelテンプレートルール

- 作成画面と提出用PDFは完全に別レイアウトにする。
- 提出用PDFにはサイドバー、プルダウン、ボタン、編集UIを出さない。
- 提出用帳票の見た目は `templates/御見積書_提出用テンプレート.xlsx` を基準にする。
- 元テンプレートを直接破壊せず、必要なデータだけを決まったセルへ流し込む。
- `outputs/` は生成物として Git 管理しない。
- Excelテンプレートと `templates/takemoto-seal.png` はPDF出力に必要なアプリ部品としてGit管理する。

## テスト・確認ルール

- 見積計算ロジック変更時は `npm.cmd test` を実行する。
- PDF出力変更時は、会社宛・個人宛、明細少数・複数ページ、ロゴ・印鑑ありなしを確認する。
- PowerShell で `npm test` が失敗する場合は `npm.cmd test` を使う。
